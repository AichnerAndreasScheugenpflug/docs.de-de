# <a name="conversion-from-type1-to-type2-cannot-occur-in-a-constant-expression-used-as-an-argument-to-an-attribute"></a><span data-ttu-id="76b4a-102">Konvertierung von "\<Typ1 >', '\<Typ2 >" kann nicht in einem konstanten Ausdruck als Argument für ein Attribut verwendet auftreten</span><span class="sxs-lookup"><span data-stu-id="76b4a-102">Conversion from '\<type1>' to '\<type2>' cannot occur in a constant expression used as an argument to an attribute</span></span>
<span data-ttu-id="76b4a-103">Ein Ausdruck, der für ein Attributargument verwendet, die in einen Datentyp der entsprechenden attributparameters unterscheidet ausgewertet wird, und Visual Basic lässt die erforderliche typkonvertierung für Attributargumente nicht zu.</span><span class="sxs-lookup"><span data-stu-id="76b4a-103">An expression used for an attribute argument evaluates to a data type different from that of the corresponding attribute parameter, and Visual Basic does not allow the required type conversion for attribute arguments.</span></span>  
  
 <span data-ttu-id="76b4a-104">Ein Attribut stellt Metadaten für das Element bereit, auf das es angewendet wird, und der Compiler muss zur Kompilierzeit in der Lage sein, alle Metadaten zu erstellen.</span><span class="sxs-lookup"><span data-stu-id="76b4a-104">An attribute provides metadata for the element it is applied to, and the compiler must be able to construct all the metadata at compile time.</span></span> <span data-ttu-id="76b4a-105">Daher muss jedes Attribut Werte verwenden, die zur Kompilierzeit konstant sind, und jedes Attributargument muss als ein zur Kompilierzeit konstanter Wert ausgewertet werden.</span><span class="sxs-lookup"><span data-stu-id="76b4a-105">For this reason, every attribute must use values that are constant at compile time, and therefore every attribute argument must evaluate to a compile-time constant value.</span></span>  
  
 <span data-ttu-id="76b4a-106">Bei bestimmten Typkonvertierungen können keine Werte erzeugt werden, die zur Kompilierzeit konstant sind.</span><span class="sxs-lookup"><span data-stu-id="76b4a-106">Certain type conversions cannot produce values that are constant at compile time.</span></span> <span data-ttu-id="76b4a-107">Ob beispielsweise `String` in `Double` oder `Date` konvertiert wird, hängt von der Gebietsschemaeinstellung zur Laufzeit ab.</span><span class="sxs-lookup"><span data-stu-id="76b4a-107">For example, converting a `String` to a `Double` or a `Date` depends on the locale setting at run time.</span></span> <span data-ttu-id="76b4a-108">Andere Konvertierungen (z. B. die Konvertierung eines Array eines abgeleiteten Typs in ein `Object`-Array) verursachen eine Vielzahl von Problemen, durch die der Compiler sie für Attributargumente nicht zulassen kann.</span><span class="sxs-lookup"><span data-stu-id="76b4a-108">Other conversions, such as an array of a derived type to an array of `Object`, present a variety of problems that do not permit the compiler to allow them on attribute arguments.</span></span>
